# GeoQuest

GeoQuest is a location-based Android application inspired by geocaching, developed as part of a university group project. Working in an Agile team, I contributed to designing and implementing features that combined GPS, QR code scanning, and mobile sensors to create an interactive treasure hunt experience. The application encourages users to explore real-world locations, discover virtual caches, and track their progress.

## Project Objectives

The project aimed to create an engaging location-based game that combines mobile technologies with outdoor exploration. The main objectives were:

- Develop a location-based treasure hunt application for Android devices.
- Display caches on an interactive map using GPS and location services.
- Allow users to discover and unlock caches by scanning QR codes.
- Integrate mobile sensors, including a digital compass, to assist navigation.
- Enable users to upload photos of discovered caches.
- Store and retrieve user progress using persistent cloud storage.
- Create a clean and intuitive mobile interface following Material Design principles.
- Develop the application using an Agile workflow with collaborative version control.

## Technologies Used

- React Native for developing the Android mobile application.
- React Native Paper for building a consistent Material Design user interface.
- Expo Camera for taking photos and scanning QR codes.
- Expo Sensors for accessing the device's magnetometer to implement the compass.
- React Native Maps for displaying caches and the user's location.
- react-native-qrcode-svg for generating QR codes.
- Cloudinary for storing uploaded images and generating public image URLs.
- Git & GitHub for version control and collaborative Agile development.

## User Interface

The application was designed using React Native Paper to provide a clean and consistent Material Design interface. The layouts focused on simplicity and ease of navigation, ensuring users could quickly access the map, compass, camera, and cache information while exploring.

Special attention was given to presenting location information clearly and keeping interactions intuitive, allowing users to focus on the treasure hunt experience rather than learning a complex interface.

Throughout development, the interface evolved through regular Agile sprint reviews and team feedback, ensuring the final design remained user-friendly and met the project's objectives.
